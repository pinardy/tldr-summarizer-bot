A day in the life of a digest

21:00 SGT · fetch
🗞️

The paper round

GitHub Actions wakes as the day's TLDR issues publish, fetches all six from tldr.tech, and parses every story — headline, blurb, link — while filtering out sponsors, self-promo, and tracking parameters.

+30 seconds · merge
🦉

The editor's desk

One LLM call (opencode's deepseek-v4-flash, covered by a Go subscription) merges ~75 stories into the 10–15 that matter, deduplicates cross-newsletter coverage, and groups them into themed sections. Every link is validated against the source — no hallucinated URLs.

+90 seconds · deliver
📬

The evening edition

The digest lands in Telegram as one tidy message, gets archived to the public website, and the dedup ledger is committed back to the repo — so no issue is ever delivered twice, even across retries and reruns.

2

Two ways to get your news

📮 The subscription

Scheduled push — it comes to you

  • Runs at 21:00 (same-day delivery) and 09:00 SGT (catch-up) on GitHub Actions' free tier — no machine of yours needs to be awake.
  • Weekends and holidays skip themselves: no issue published, nothing sent.
  • If the AI is ever unavailable, delivery degrades gracefully to full per-newsletter story lists — the digest always arrives.
  • Every digest is archived to a static GitHub Pages site, browsable by date.
cron→ parse→ merge→ Telegram→ archive

💬 The newsroom hotline

Real-time pull — you ask, it answers

  • A Cloudflare Worker (free tier) receives your Telegram messages by webhook, instantly.
  • /digest builds today's briefing on demand — cached, so repeats answer in seconds.
  • /news kubernetes or any free-form question gets answered from today's actual stories, with conversation memory for follow-ups.
  • Private by design: secret-token webhook auth, and only one chat ID ever gets an answer.
message→ Worker→ KV cache→ answer

The fine print (it's all good news)

Wanted: no servers

The whole system is scheduled compute and edge functions. Nothing to patch, restart, or pay for at 3am.

Lost & found: state

Actions runners are ephemeral, so the dedup ledger is committed back to git after every run — version-controlled memory.

For sale: nothing

GitHub Actions free tier, Cloudflare Workers free tier, and an already-paid opencode Go subscription. Marginal cost: zero.

Situations vacant: none

Failures self-heal: fetch errors skip a newsletter, AI errors fall back to parsed digests, total failure sends a Telegram warning.
